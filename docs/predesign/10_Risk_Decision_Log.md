# 10. Risk & Decision Log

## 1. 关键风险

| 风险 | 等级 | 缓解措施 | Owner |
|---|---|---|---|
| Agent 输出质量不足 | 高 | 建评测集、Prompt 版本化、低信心标记、人工反馈 | AI 工程 |
| RAG 引用不准确 | 高 | 引用 ID 校验、检索测试、人工审核 Chunk、废弃状态 | AI/后端 |
| 多 Agent 编排不稳定 | 高 | 状态机、超时、重试、最大轮次、部分报告 | 后端 |
| 企业数据合规 | 高 | 私有部署选项、脱敏、租户隔离、审计 | 架构/安全 |
| UI 信息量过大 | 中 | 三栏布局、渐进展开、状态明确 | antigravity |
| 开发范围膨胀 | 高 | MVP 锁定核心闭环，P1/P2 延后 | 产品 |
| 第三方集成拖慢 MVP | 中 | Adapter + mock，真实集成延后 | 后端 |

## 2. 已建议决策

| 决策 | 建议 | 理由 |
|---|---|---|
| 仓库结构 | monorepo | 共享类型和上下文，适合 AI 辅助开发 |
| 架构形态 | 模块化单体 + Worker | MVP 复杂度可控，后续可拆服务 |
| 向量库 | pgvector 起步 | 降低部署复杂度 |
| 会议模式 | Round-Robin + Free Debate | 覆盖稳定基线和差异化体验 |
| 信心指数 | 规则版起步 | 无需等待训练数据 |
| 导出 | HTML/MD 起步，PDF/DOCX 后补 | 避免排版阻塞主闭环 |
| 推演引擎 | P1 | 技术难度高，不应阻塞 MVP |

## 3. 待产品确认

1. MVP 是否必须支持真实 PDF/DOCX 导出？
2. 首个试点行业/场景是技术方案评审，还是泛方案评审？
3. 是否允许使用公有云模型 API 处理企业文档？
4. 报告是否需要“人工终审签字发布”？
5. 角色预置 Prompt 是否由领域专家共同审核？
6. Action Items 是否必须首版打通 Jira？
7. 是否需要支持私有化部署作为 MVP 前提？
