# 01. Product Design Blueprint

## 1. 产品一句话定义

PrismReview 是面向企业方案评审场景的多 Agent 智能评审中枢：用户提交方案后，系统自动组建跨领域 AI 评审委员会，以结构化会议形式输出可追溯、可执行、可沉淀的评审报告。

## 2. 产品设计原则

1. **辅助决策，不替代决策**：系统输出建议、风险和行动项，最终判断权保留给人。
2. **多视角优先于单点最优**：价值来自 CTO/CFO/PMO/Compliance/User Advocate 等角色的交叉审查。
3. **白盒可信**：每条意见都应说明来源、依据、信心指数和可执行建议。
4. **流程闭环**：从提交方案、组局、会议、报告、Action Items 到知识沉淀形成闭环。
5. **企业可控**：角色、知识库、权限、审计、数据边界必须可配置、可治理。

## 3. 目标用户与核心任务

| 用户 | 核心任务 | 高频页面 |
|---|---|---|
| 技术负责人 / 架构师 | 快速获得技术、交付、合规多维评审 | 发起评审、会议流、报告 |
| 产品经理 | 在正式评审前进行预评审和盲区扫描 | 发起评审、历史对比、报告 |
| PMO | 识别排期、资源、依赖风险 | 报告、Action Items |
| 合规 / 法务 | 审查法规、隐私、许可证风险 | 知识库、报告、角色配置 |
| 企业管理者 | 看全局风险、采纳率、审计 | 管理后台、数据仪表盘 |
| 系统管理员 | 配置角色、知识库、权限 | 角色管理、知识库、权限 |

## 4. MVP 范围

### 4.1 必须包含

- 登录后主工作台。
- 发起评审：文档上传或文本粘贴、目标描述、会议模式选择。
- Chairman 方案诊断书：领域标签、风险维度、推荐角色、邀请理由。
- 角色调整：增删角色、调整权重、快速模式。
- 会议引擎：轮流发言制、自由辩论制。
- 人机协同：暂停/举手打断/注入新条件。
- 白盒意见：风险等级、证据引用、信心指数、建议。
- 报告：六章结构、Action Items、导出占位接口。
- 角色管理：预置 5 角色、自定义角色、版本管理。
- 知识库管理：上传、解析、分块、索引、检索测试、条目审核状态。
- 权限基础：租户、部门、用户、角色权限。
- 审计与事件：关键操作留痕，业务事件埋点。

### 4.2 MVP 可降级

- PDF/DOCX 导出可先生成 Markdown/HTML，再接导出服务。
- 信心指数可先规则化计算，后续引入训练模型。
- RAG 增量更新可先整文重建索引，保留接口抽象。
- 第三方集成先提供 Webhook/Jira mock adapter。

### 4.3 非 MVP

- 盲审共识、红蓝对抗完整流程。
- 因果链图与假设推演。
- 角色市场与组织知识自动沉淀。
- 复杂计费、区域化部署、多语言完整覆盖。

## 5. 核心模块边界

| 模块 | 负责什么 | 不负责什么 |
|---|---|---|
| Review Workspace | 发起、组局、会议、报告 | 角色模板的深度编辑 |
| Agent Role Center | 角色定义、版本、知识挂载 | 模型推理执行 |
| Knowledge Center | 文档、Chunk、向量索引、检索测试 | 业务报告生成 |
| Orchestration Engine | Chairman、会议模式、任务状态流转 | 前端 UI 渲染 |
| Report Center | 报告结构化、导出、分享 | 原始模型编排 |
| Action Center | 行动项、状态、外部推送 | 项目管理工具完整替代 |
| Admin Console | 权限、权重、审计、基础配置 | 业务评审执行 |

## 6. 产品成功指标

| 指标 | MVP 目标 | 后续目标 |
|---|---:|---:|
| 方案诊断生成耗时 | ≤ 10 秒展示首屏 | ≤ 5 秒 |
| 从确认组局到首个 Agent 输出 | ≤ 5 秒进入执行状态 | ≤ 3 秒 |
| 报告生成时间 | 轮流发言 ≤ 30 秒 | 所有模式 ≤ 60 秒 |
| 建议采纳率 | 可采集反馈 | ≥ 65% |
| 低信心意见人工确认入口 | 100% 覆盖 | 形成质量学习闭环 |
| 多租户隔离 | API 层强校验 | 通过渗透测试 |

## 7. 关键产品决策

1. MVP 以“轮流发言制”作为稳定基线，“自由辩论制”作为差异化体验。
2. Chairman 不直接给最终裁决，只负责诊断、调度、汇总和冲突归纳。
3. Agent 输出必须结构化，不允许只有自然语言长文。
4. 所有 AI 输出均保留原始 trace、模型参数、上下文摘要和引用证据。
5. 企业版默认开启审计日志与租户隔离，不作为高级功能延后。

## 8. 开放问题

- 首批目标行业是否限定在技术/产品评审，还是同时覆盖投资/法务？
- 模型供应商选择：单模型、多模型路由，还是企业私有模型接入？
- 文档是否允许离开企业环境进入第三方模型 API？
- 是否需要“人工终审签字”作为报告发布前置条件？
- MVP 是否必须支持 PDF/DOCX 原生导出？
