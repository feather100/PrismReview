# 09. Test Strategy and Acceptance

## 1. 测试目标

- 验证核心评审闭环可用。
- 验证 AI 输出结构化、可追溯、低幻觉风险。
- 验证多租户和权限隔离。
- 验证 RAG 检索和引用链路。
- 验证长任务、超时、断网恢复等异常态。

## 2. 测试分层

| 层级 | 范围 | 工具建议 |
|---|---|---|
| Unit | 权限、状态机、信心指数、schema 校验 | Jest/Pytest |
| Integration | API + DB + Queue + Worker | Testcontainers |
| E2E | 浏览器核心流程 | Playwright |
| AI Eval | Prompt/RAG/Agent 输出质量 | 固定评测集 + 人工评分 |
| Security | 租户隔离、权限、文件上传 | OWASP checklist |
| Performance | 检索、诊断、报告、并发评审 | k6/Locust |

## 4. 核心 E2E 用例

### E2E-01 创建并完成轮流发言评审

1. 用户登录。
2. 上传方案文档。
3. 填写目标。
4. 生成诊断书。
5. 接受推荐角色。
6. 启动 Round-Robin。
7. 等待 3 个 Agent 输出。
8. 生成报告。
9. 验证报告包含风险矩阵和 Action Items。

### E2E-02 角色调整

1. 进入诊断书。
2. 删除 CFO。
3. 添加自定义角色。
4. 调整权重为 1.5x。
5. 确认组局。
6. 验证会议角色与调整一致。

### E2E-03 知识库上传与检索测试

1. 管理员上传 PDF/MD。
2. 等待 ready。
3. 查看 Chunk。
4. 标记一个 Chunk 为 approved。
5. 执行检索测试。
6. 验证返回 Top-5 和来源信息。

### E2E-04 人机干预

1. 启动评审。
2. 等待首个 Agent 发言。
3. 点击举手打断。
4. 输入新条件。
5. 验证会议暂停后继续。
6. 验证报告记录干预条件。

### E2E-05 权限隔离

1. A 部门管理员创建私有知识库。
2. B 部门管理员尝试访问。
3. 验证页面显示无权限。
4. API 返回 403。

## 5. AI 输出质量评测

### 5.1 评分维度

| 维度 | 评分 |
|---|---|
| 风险覆盖 | 0-5 |
| 建议可执行性 | 0-5 |
| 引用准确性 | 0-5 |
| 幻觉控制 | 0-5 |
| 角色一致性 | 0-5 |
| 结构化合规 | pass/fail |

### 5.2 阈值

- 结构化输出通过率 ≥ 98%。
- 引用不存在率 = 0。
- 严重幻觉率 ≤ 2%。
- 人工评分平均 ≥ 3.5/5。

## 6. 性能测试

| 场景 | 目标 |
|---|---|
| RAG 检索 | P95 ≤ 3 秒 |
| 诊断书首屏 | P95 ≤ 10 秒 |
| 报告生成 | Round-Robin P95 ≤ 30 秒 |
| 并发评审 | 基础版 10 场同时进行 |
| SSE 连接 | 断连后 5 秒内自动重连 |

## 7. 安全测试

- 跨租户 review id 枚举。
- 跨租户 document/chunk 访问。
- 部门私有知识库越权。
- 文件上传恶意类型伪装。
- 大文件与 zip bomb 防护。
- 审计日志篡改权限。
- 下载链接过期校验。

## 8. 回归测试触发条件

- Prompt 模板修改。
- 模型供应商或模型版本修改。
- RAG chunking 或 embedding 策略修改。
- 权限逻辑修改。
- Review 状态机修改。
- 报告 schema 修改。
