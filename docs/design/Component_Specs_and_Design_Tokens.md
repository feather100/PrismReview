## 1. Ant Design 风格 Token

### 1.1 色彩系统

```less
// ─── Primary Brand ───
@primary-color: #4F46E5;           // Indigo-600 — 主色，按钮/链接/激活态
@primary-hover: #6366F1;           // Indigo-500
@primary-active: #4338CA;          // Indigo-700
@primary-bg: #EEF2FF;              // Indigo-50   — 浅色背景

// ─── AI Accent ───
@ai-accent: #7C3AED;               // Violet-600  — Agent 正在思考、诊断书高亮
@ai-accent-soft: #F5F3FF;          // Violet-50

// ─── Risk Levels ───
@risk-high: #DC2626;               // Red-600     — Tag、边框、图标
@risk-medium: #EA580C;             // Orange-600
@risk-low: #D97706;                // Amber-600
@risk-info: #6B7280;               // Gray-500

// ─── Neutral / Surface ───
@bg-page: #F3F4F6;                 // Gray-100    — 页面背景
@bg-card: #FFFFFF;                 // 卡片背景
@bg-section: #F9FAFB;             // Gray-50     — 区域分隔
@border-default: #E5E7EB;         // Gray-200
@border-hover: #D1D5DB;           // Gray-300

// ─── Text ───
@text-primary: #111827;            // Gray-900    — 正文
@text-secondary: #4B5563;          // Gray-600    — 次要信息
@text-tertiary: #9CA3AF;           // Gray-400    — 占位符/禁用
@text-inverse: #FFFFFF;

// ─── Semantic ───
@success: #16A34A;                 // Green-600   — 完成、就绪
@warning: #D97706;                 // Amber-600   — 警告
@error: #DC2626;                   // Red-600     — 错误
@info: #2563EB;                    // Blue-600    — 信息
```

#### 1.1.1 色彩使用矩阵

| 用途 | Token | 使用场景 |
|---|---|---|
| 主按钮背景 | `@primary-color` | 生成诊断书、确认组局、开始评审 |
| 链接 | `@primary-color` | 文本链接、Tab 激活 |
| Agent 思考态 | `@ai-accent` | 发言中呼吸灯、诊断书雷达图 |
| 严重风险 Tag | `@risk-high` | Tag 背景/边框，告警横幅 |
| 页面背景 | `@bg-page` | Layout 的 content 背景 |
| 卡片背景 | `@bg-card` | Card 容器 |
| 一级文本 | `@text-primary` | 标题、意见内容 |
| 二级文本 | `@text-secondary` | 描述、时间戳、元数据 |
| 成功/完成 | `@success` | 发言完成 Tag、进度绿点 |

### 1.2 间距系统

采用 Ant Design 的 `@spacing` 8px 栅格：

```less
@spacing-xxs: 4px;    // 极小间距 — Tag 内边距、图标与文字间
@spacing-xs: 8px;     // 基础间距 — 水平排列元素间距
@spacing-sm: 12px;    // 紧凑间距 — Card 内元素间距
@spacing-md: 16px;    // 标准间距 — Card padding
@spacing-lg: 24px;    // 大间距 — Section/模块间距
@spacing-xl: 32px;    // 更大间距 — 页面主要区块间隔
@spacing-xxl: 48px;   // 页面顶部/底部留白

// 布局约束
@layout-header-height: 56px;
@layout-sidebar-width: 240px;
@layout-content-max: 1440px;
@layout-content-padding: 24px;
```

### 2.1 页面布局

```
┌─── 方案诊断书 ──────────────────────────────────────────┐
│ [路径: 工作台 > 发起评审 > 方案诊断书]                   │
│                                                          │
│ ┌─ PageHeader ────────────────────────────────────────┐ │
│ │ 标题: "方案诊断书: {review.title}"                    │ │

│                                                          │
│ ┌─ 两栏布局 ─────────────────────────────────────────┐ │
│ │ Left (60%)                    Right (40%)           │ │
│ │ ┌─ SummaryCard ─────┐        ┌─ TeamCard ────────┐│ │

```

### 2.2 组件清单

| 组件 | Ant Design 映射 | 说明 |
|---|---|---|
| PageHeader | `@ant-design/pro-components` 或自建 | 标题 + 面包屑 + 操作按钮 |
| SummaryCard | `Card` | 方案摘要文本 + Tag 标签列表 |
| RadarCard | `Card` + ECharts/Chart.js | 雷达图 + 置信度百分比 |
| TeamCard | `Card` + `List` | Agent 推荐卡片列表 |
| AgentCardItem | `Card` 内嵌 | 头像/代号 + 名称 + 权重Slider + 推荐理由 + [移除] |
| RoleSelector | `Select` + `Modal` | 添加角色的下拉选择器 |
| WeightSlider | `Slider` | 权重调整，总和约束为 100 |
| RiskTag | `Tag` | 风险维度标签（性能、安全、成本等） |
| DiagnosisConfidence | `Progress` (环形) | 诊断置信度 0-100% |
| EmptyPlaceholder | `Empty` | 无数据/空态占位 |
| Spin | `Spin` | 加载中覆盖层 |

### 2.3 状态定义

#### 2.3.1 空态 (Empty)

**触发条件**：从未生成诊断书 — 用户直接导航至此页面。

```
┌─ 方案诊断书 ──────────────────────────────────────────┐
│                                                          │
│            🔍 暂无诊断数据                                │
│            请先发起评审以生成方案诊断书                    │
│                                                          │
│              [ 去发起评审 ]   ← 跳转 /reviews/new         │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

- 页面 Header 不显示子标题和操作按钮。
- 内容区居中显示 `Empty` 组件，主按钮跳转发起评审页。
- **API 行为**：GET `/api/reviews/{id}/diagnosis` 返回 `null`。

#### 2.3.2 加载态 (Loading)

**触发条件**：用户点击"生成方案诊断书"后，Chairman 诊断任务进行中。

```
┌─ 方案诊断书 ──────────────────────────────────────────┐
│                                                          │
│  🕐 诊断生成中 (用时约 5-10 秒)                          │
│                                                          │
│  ┌─ 骨架屏 (Skeleton) ─────────────────────────────┐   │
│  │ ┌────────────────┐   ┌──────────────────────┐   │   │
│  │ │ ████████████    │   │ ████████████████████  │   │   │
│  │ │ ████████████    │   │ ████████████████████  │   │   │
│  │ │ ████████████    │   │ ████████████████████  │   │   │
│  │ └────────────────┘   └──────────────────────┘   │   │
│  └──────────────────────────────────────────────────┘   │
│                                                          │
│  ⚡ 诊断进度: ████████░░░░ 65%                           │
│    当前: 正在分析方案领域标签...                          │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

- 使用 `Skeleton` 组件（而非 `Spin`）模拟最终布局骨架。
- 底部显示 `<Progress>` 条 + 阶段文字（"正在分析方案领域标签…" → "正在匹配评审角色…" → "正在评估风险维度…"）。
- 操作按钮禁用。
- **超时保护**：15 秒后显示 "诊断超时" 引导手动组局（见错误态）。

#### 2.3.3 错误态 (Error)

**触发条件**：诊断 API 失败 / 超时 / 解析失败。

```
┌─ 方案诊断书 ──────────────────────────────────────────┐
│                                                          │
│  ❌ 诊断生成失败                                          │
│  ┌─ Alert (type=error) ─────────────────────────────┐   │
│  │ 诊断服务暂时不可用，原因: {error.message}         │   │
│  │ 方案诊断需要文档解析 + 语义检索，请检查文档格式。  │   │
│  └──────────────────────────────────────────────────┘   │
│                                                          │
│  您仍然可以手动组局：                                     │
│                                                          │
│  ┌─ 手动组局面板 ─────────────────────────────────┐   │
│  │ [方案摘要] (可编辑文本域)                        │   │
│  │ [选择角色] (多选 Select)                         │   │
│  │ [设置权重] (Sliders)                             │   │
│  │              [ 手动确认并开始评审 ]              │   │
│  └──────────────────────────────────────────────────┘   │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

- 顶部 `Alert` 展示错误信息，支持 `type="error"` 和可关闭。
- 诊断失败不阻塞业务：降级为 **手动组局面板**，允许用户输入摘要、选择角色、设置权重。
- **重试按钮**：Alert 内嵌 [重试] 按钮，重新调用诊断 API。

#### 2.3.4 权限态 (Forbidden)

**触发条件**：当前用户对该评审无诊断书查看权限 / 跨租户访问。

```
┌─ 方案诊断书 ──────────────────────────────────────────┐
│                                                          │
│  🔒 权限不足                                              │
│  ┌─ Alert (type=warning) ───────────────────────────┐   │
│  │ 您没有查看该方案诊断书的权限                        │   │
│  │ 请联系管理员或项目创建者申请访问权限。              │   │
│  └──────────────────────────────────────────────────┘   │
│                                                          │
│  [ 申请访问 ]  [ 返回工作台 ]                             │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

- 不展示任何诊断数据。
- 底部两个按钮："申请访问"（调用权限申请接口）和"返回工作台"。
- **API 行为**：GET `/api/reviews/{id}/diagnosis` 返回 403 `TENANT_ISOLATION_VIOLATION` 或 `FORBIDDEN`。

#### 2.3.5 断连态 (Disconnected)

**触发条件**：SSE/WebSocket 连接中断（实时诊断流场景）。

```
┌─ 方案诊断书 ──────────────────────────────────────────┐
│                                                          │
│  🔴 诊断结果                        [网络已断开]         │
│  ┌─ Alert (type=warning) ───────────────────────────┐   │
│  │ ⚠️ 网络连接已断开，诊断结果可能不完整。             │   │
│  │ 正在尝试自动重连...                                │   │
│  └──────────────────────────────────────────────────┘   │
│                                                          │
│  (已加载的诊断内容展示 — 只读)                           │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

- 顶部 Banner + 重连状态提示。
- 已加载的数据保持展示，但标记为"可能不完整"。
- 用户可在此状态下操作（如确认组局），但保存前提示网络状态。

### 2.4 状态流转图

```
[页面加载]
   │
   ├──→ 诊断书存在? ──→ 无 ──→ [空态: 引导发起评审]
   │
   └──→ 有 ──→ 诊断书数据 ──→ [正常态]
                 │
                 ├──→ 网络断开 ──→ [断连态 Banner]
                 ├──→ API 403  ──→ [权限态]
                 └──→ 重新诊断 ──→ [加载态 Skeleton]
                                    │
                                    ├──→ 成功 ──→ [正常态]
                                    └──→ 失败 ──→ [错误态: 手动组局]
```

## 3. 评审会议室 — 组件与状态规格

### 3.1 页面布局

```
┌─── 评审会议室 ──────────────────────────────────────────┐
│ [路径: 工作台 > 评审 {id}]                                │
│                                                          │
│ ┌─ MeetingHeader ──────────────────────────────────────┐ │
│ │ Tag: 🔴 进行中 | 标题 | 模式: 轮流发言 | 进度: 2/5  │ │
│ │ 操作: [✋ 打断] [⏸ 暂停] [⏹ 强制结束]              │ │
│ └─────────────────────────────────────────────────────┘ │
│                                                          │
│ ┌─ 三栏布局 ──────────────────────────────────────────┐ │
│ │ Left (20%)          Center (55%)     Right (25%)     │ │
│ │ ┌─ AgentPanel ─┐   ┌─ SpeechFlow ─┐  ┌─ ContextPanel│ │
│ │ │ CTO 🟢 完成  │   │ [发言卡片流]  │  │ 📌 方案摘要  │ │
│ │ │ CFO 🗣️发言中│   │              │  │ 📎 知识引用  │ │
│ │ │ PMO ⏳排队  │   │              │  │ ✋ 干预条件   │ │
│ │ │ Compl ⏳排队 │   │              │  │ ─────────── │ │
│ │ │ UA ⏳排队   │   │              │  │ [暂停会议]   │ │
│ │ └─────────────┘   └──────────────┘  │ [注入新条件]  │ │
│ │                                      │ [强制结束]   │ │
│ │                                      └──────────────│ │
│ └─────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 3.2 组件清单

| 组件 | Ant Design 映射 | 说明 |
|---|---|---|
| MeetingHeader | 自建 | 顶部状态栏：状态 Tag + 标题 + 模式 + 进度 + 操作按钮 |
| StatusTag | `Tag` | 会议状态（进行中/已暂停/已结束） |
| AgentPanel | `Card` | Agent 列表，每个条目含 Avatar + 名称 + 状态指示灯 |
| AgentStatusDot | 自建（CSS） | 🟢 完成 / 🗣️ 发言中 / ⏳ 排队 / ❌ 失败 / ⏰ 超时 |
| SpeechFlow | `List` + `Card` | 可滚动发言流，按时间逆序 |
| SpeechCard | `Card` | 单个 Agent 发言：Avatar + 角色名 + 风险Tag + 维度 + 内容 + 引用 + 信心指数 |
| ContextPanel | `Card` | 右侧面板固定宽度，含方案摘要、引用卡片、干预记录 |
| InterventionForm | `Modal` + `Form` | 注入新条件的表单（文本输入 + 影响范围选择） |
| CitationCard | `Card` | 引用知识条目：文档名 + 页码 + 片段预览 |
| ConfidenceBadge | `Badge` | 信心指数展示（高/中/低三色） |

### 3.3 状态定义

#### 3.3.1 空态 (Empty)

**触发条件**：评审已创建但尚未开始（status=draft / ready），用户提前进入会议室页面。

```
┌─ 评审会议室 ──────────────────────────────────────────┐
│                                                          │
│  💠 评审尚未开始                                          │
│                                                          │
│  ┌─ SummaryCard ─────────────────────────────────────┐   │

- 右侧 ContextPanel 折叠。
- 若用户无权限启动（非创建者），按钮禁用并提示"请联系发起人开始评审"。

#### 3.3.2 加载态 (Loading)

**触发条件**：正在加载评审历史消息 / 正在编排下一个 Agent 发言。

```
└─ 评审会议室 ──────────────────────────────────────────┐
│                                                          │
│ ┌─ MeetingHeader ────────────────────────────────────┐   │
│ │ Tag: 🟡 连接中 | 标题 | 模式 | 进度: -/-         │   │
│ └────────────────────────────────────────────────────┘   │
│                                                          │

```

- MeetingHeader 显示 "连接中" 状态 Tag（`@warning` 色）。
- 三栏均为 `Skeleton` 骨架屏。
- **Agent 发言间加载**：下一个 Agent 发言时，Center 区域底部显示 `Spin` + "{role_name} 正在检索知识库…" / "{role_name} 正在分析…"。

#### 3.3.3 错误态 (Error)

**触发条件**：Agent 超时 / Agent 失败 / 编排异常。

**3.3.3a Agent 超时**

```
┌─ SpeechCard ──────────────────────────────────────────┐
│ 👤 PMO                             ⏰ 已超时 (120s)     │
│ ┌─ Alert (type=warning) ───────────────────────────┐   │
│ │ PMO 未在限定时间内完成发言。                        │   │

│ └──────────────────────────────────────────────────┘   │
│                                                        │
│ 报告中将标注: ⚠️ 完整性风险 — PMO 未输出意见            │
└─────────────────────────────────────────────────────────┘
```

**3.3.3b Agent 失败**

```
┌─ SpeechCard ──────────────────────────────────────────┐
│ 👤 Compliance                        ❌ 发言失败        │
│ ┌─ Alert (type=error) ─────────────────────────────┐   │
│ │ 合规审查员发言时发生错误: {error}                  │   │
│ │ [ 重试 ] [ 跳过并继续 ]                           │   │
│ └──────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

- 超时：自动跳过，Alert 展示超时信息。
- 失败：显示错误信息 + [重试 / 跳过] 按钮。
- 右侧 AgentPanel 中对应 Agent 状态标为 ❌。
- **全局编排失败**：弹出 Modal "评审编排异常，已生成部分报告。"

#### 3.3.4 权限态 (Forbidden)

**触发条件**：用户被移除出评审 / 跨租户访问 / 权限变更。

```
┌─ 评审会议室 ──────────────────────────────────────────┐
│                                                          │
│  🔒 无权访问该评审                                        │
│                                                          │
│  ┌─ Alert (type=warning) ───────────────────────────┐   │
│  │ 您没有访问此评审会议的权限。                        │   │
│  │ 可能的原因：角色已被移除 / 评审已归档 / 权限变更   │   │
│  └──────────────────────────────────────────────────┘   │
│                                                          │
│  [ 联系管理员 ]  [ 返回工作台 ]                           │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

- 不渲染任何会议组件，不建立 WebSocket 连接。
- **API 行为**：WS 连接返回 403 / HTTP API 返回 `FORBIDDEN`。

#### 3.3.5 断连态 (Disconnected)

**触发条件**：SSE/WebSocket 连接中断（网络波动 / 服务重启）。

```
┌─ 评审会议室 ──────────────────────────────────────────┐
│                                                          │
│ ┌─ Banner (sticky) ─────────────────────────────────┐   │
│ │ ⚠️ 网络连接已断开                                     │   │
│ │ 🔄 正在自动重连... (第 2/5 次)                        │   │
│ │ [ 手动重连 ]                                         │   │
│ └────────────────────────────────────────────────────┘   │
│                                                          │
│ ┌─ (三栏保持最后已知状态，只读) ─────────────────────┐   │
│ │ AgentPanel: 状态冻结                                │   │
│ │ SpeechFlow: 最后消息停留，标记 "以下内容可能不完整" │   │
│ │ ContextPanel: 只读                                  │   │
│ └────────────────────────────────────────────────────┘   │
│                                                          │
│ 底部: "断连期间的消息将在重连后自动补齐"                  │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

- 自动重连策略：5 次，间隔 2s/4s/8s/16s/30s 递增。
- 重连成功后，Banner 消失，补发断连期间的发言卡片（标记 "补发"）。
- 重连均失败后，Banner 变红："⚠️ 无法重新连接，请刷新页面"。

### 3.4 状态流转图

```
[用户进入会议室]
   │
   ├── 评审状态 = draft/ready ──→ [空态: 开始按钮]
   │                                 │
   │                                 └── 点击开始 ──→ API: POST /start
   │                                                  │
   │                                                  └── 成功 ──→ [正常态 三栏]

   ├── 评审状态 = running ──→ [正常态 三栏]
   │                            │
   │                            ├── WS 断开 ──→ [断连态]
   │                            │                ├── 重连成功 ──→ 补发消息
   │                            │                └── 均失败 ──→ 提示刷新
   │                            │
   │                            ├── Agent 超时 ──→ 跳过 + Alert
   │                            ├── Agent 失败 ──→ Alert + [重试/跳过]
   │                            ├── 编排异常 ──→ Modal + 部分报告
   │                            │
   │                            └── 用户打断 ──→ [暂停] → 注入条件 → 继续

```

```

- 已加载的部分保持可读。
- 未加载的章节显示 `Spin` + "等待网络恢复…"。
- 导出按钮禁用并 Tooltip "网络断开时暂不可用"。

## 5. 跨页面状态汇总矩阵

| 状态 | 方案诊断书 | 评审会议室 | 评审报告页 |
|---|---|---|---|
| **空态** | 引导发起评审 | 评审概要 + 开始按钮 | 返回会议室引导 |
| **加载态** | Skeleton 骨架 + Progress 进度 | 三栏 Skeleton + 发言间 Spin | 六章 Skeleton + 章节进度 |
| **错误态** | 降级手动组局面板 | Agent 超时跳过 / 失败重试 / 编排异常 Modal | 重新生成 / 完整性风险 Banner |
| **权限态** | 403 Alert + 申请访问 | 403 Alert + 联系管理员 | 403 Alert + 部门隔离说明 |
| **断连态** | Banner + 已加载内容只读 | Sticky Banner + 自动重连 + 补发机制 | Banner + 已加载章节可读 + 导出禁用 |
| **超时保护** | 15s 诊断超时降级 | 120s Agent 超时自动跳过 | 60s 报告超时显示部分章节 |
